Merge sort
Merge sort is advanced sorting algorithm It uses the divide-and-conquer strategy
Ideal divide-and-conquer is to have a balanced partition
Drawback of quicksort is that we can not guarantee the partition is balanced
We want to have a perfect balanced partition at all cost. T(n) = 2T(n/2) + O(n) ==> T = O(n log(n))

We scarify the nice property that any element in 1st part <= 2 any in 2nd part, to exchange a perfect balanced partition
Mergesort: (1) partition perfectly (2) recursively merge the subsets (3) perform merge process
During merge process, every comparison will result in an element move Each move costs <= a comparison therefore Running time of merge = O(n)
Storage cost: extra array to put merged elements. This contrasts the sort-in-place of quicksort
